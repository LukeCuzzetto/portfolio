About Me

Hey, my name is Luke Cuzzetto, a recent graduate from Thompson Rivers University in Kamloops, British Columbia. I graduated with a Bachelor of Computer Science in December 2024 and am pursuing a career in Game Development. I am Gamer to the core, passionate about creating beautiful, immersive games and gaming experiences.

Programming Languages: C#, C++, GDScript, Python, Java, JavaScript, PHP, MySQL, HTML, CSS, AJAX

Game Engines and Tools: Unity, Unreal Engine, Godot, Blender, Git/GitHub.

From the time I was 10 years old, I was playing games on my family's computer in the living room, immersing myself in the stories and worlds built by past game developers. This early exposure to games created a safe haven for me while growing up. Gaming is where all us Nerds go to get away and enjoy our time.

As I grew older, I began to wonder how these games were made. I started to learn about programming and game development after high school, and I was hooked. I knew that I wanted to be a game developer and create my own worlds and stories so that other people like me can sink hours and hours into a game with no remorse, just like I have many, many times.

My Projects

Breach and Clear

This is my on-going 3D Tacticle FPS shooter that I have been working on since graduating university in December, 2024. This project was mainly motivated by games like Escape from Tarkov and Insurgency: Sandstorm.

I wanted to learn how to implement a game development tool (the KINEMATION FPS Animation tool), as game development companies use tools that are built and implemented like this all the time, and I have never done this before.

In the preview it shows the working procedural animations working as intended as well as the basic idea of the game. A breach and clear game, where a player would go through rooms to clear out the enemies while saving innocent civilians. This is my current work in progress, but the animations look too good to not show.

Colossal Brew

This project was started during an IEEE Game Development Seminar in the summer of 2024, in hopes of being submitted to the Pirate Games Gamejam.

This was a 2D project done in Godot. I was in a team of 4, with 2 other students and a Professor from my University acting as our Lore creator. This 2D platformer took place on moving "colossi" that the player would have to climb. These acted as the levels for the player. Inside these levels the player would find enemies and items to collect and bring back to their Home Base.

I worked on implementing core mechanics, such as combat and movement to the game, as well as helped create the initial Game Loops and Game Pillars for the game.

Procedural Dungeon Creator

This was a project done for my Procedural Generation class during my third year at school. This 2D Dungeon generator used multiple algorithms to create a dungeon with connecting corridors in real time. I also scripted a Enemy Spawner Agent which controlled the amount and quality of enemies spawning, which was just another element of the Procedural Content.

This was done in Unity, and taught me how to implement algorithms to create unique level design that would be excellent in any kind of Roguelike game, which was the main idea.

In the preview video, it shows the dungeon generator running, with every click of the button the level would reset and spawner would reset and a new dungeon would be created.

Aim Trainer

This is the first Game Development project I did. Being heavily addicted to Counter Strike for many years, I thought "Why not make something I can use to make me better?", so I created this simple Aim Trainer for me to use.

This simple game had a "start" button (the orange stick) that the player would interact with that would start a timed trial, which would then go for a set amount of time and count the number of targets hit during that time. Once finished, it would reset and the player would go again.

This introduced me to the basic controls and programming practices inside Unity's engine, such as Aiming, 3D Movement, Particle Systems, Raycasts, Instantiating objects and Audio Sources.
